# 🔌 Firebase 설정 가이드 (약 15분)

> 이 작업은 **여러분이 직접** 하셔야 합니다. 구글 계정이 필요하고, 제가 대신 로그인할 수 없기 때문입니다.
> 끝나면 `firebase-config.js` 파일 하나만 채워집니다. 그 외에는 아무것도 안 건드립니다.

---

## 왜 필요한가요?

학생 50명이 **각자의 기기**를 씁니다. 지금까지처럼 브라우저 안(localStorage)에만 저장하면, **각 기기에 그 학생 데이터만 남습니다.** 선생님 컴퓨터에는 50명의 데이터가 존재하지 않으므로 현황판이 영원히 빈 화면이 됩니다.

기기를 넘나드는 데이터에는 **중간에 데이터를 보관해 줄 서버**가 반드시 필요합니다. Firebase가 그 서버 역할을 무료로 해줍니다.

---

## 1단계 — Firebase 프로젝트 만들기

1. https://console.firebase.google.com 접속 → 구글 계정으로 로그인
2. **「프로젝트 추가」** 클릭
3. 프로젝트 이름: `jarada` (아무거나 괜찮습니다)
4. **Google 애널리틱스: 사용 안 함**으로 끄세요 → 「프로젝트 만들기」

> 애널리틱스를 켜면 설정 단계가 늘어나기만 하고 이 앱엔 쓸모가 없습니다.

---

## 2단계 — Realtime Database 만들기 ⚠️ 가장 중요

좌측 메뉴에서 **빌드 → Realtime Database** → **「데이터베이스 만들기」**

> ⚠️ **"Firestore Database"가 아니라 "Realtime Database"입니다.** 이름이 비슷해서 초보자가 가장 많이 틀리는 지점입니다. 잘못 고르면 코드가 전혀 동작하지 않습니다.

1. 위치: `싱가포르(asia-southeast1)` 선택 (한국에서 가장 빠릅니다)
2. 보안 규칙: **「테스트 모드에서 시작」** 선택 → 「사용 설정」

---

## 3단계 — 보안 규칙 확인 (대개 그대로 두면 됩니다)

**규칙** 탭으로 가서 확인하세요. **테스트 모드**로 만들었다면 이렇게 나옵니다.

```json
{
  "rules": {
    ".read":  "now < 1786719600000",  // 2026-8-15
    ".write": "now < 1786719600000"   // 2026-8-15
  }
}
```

✅ **이게 정상입니다. 바꾸지 마세요.** "적힌 날짜까지는 누구나 읽고 쓸 수 있다"는 뜻이고, 그 날짜는 **만든 날로부터 30일 뒤**로 자동 설정됩니다.

⚠️ **만약 `false` 로 되어 있다면** 잠금 모드로 만든 것입니다. 앱이 아무것도 못 읽습니다. 아래처럼 바꾸고 **「게시」** 를 누르세요.

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

> **📅 30일 뒤에는 앱이 멈춥니다.** 적힌 날짜가 지나면 이 화면에 다시 와서 날짜를 미루거나 `true` 로 바꾸세요. 발표가 그 안이라면 신경 쓸 필요 없습니다.
>
> ⚠️ 어느 쪽이든 **"아무나 읽고 쓸 수 있다"** 는 뜻입니다. 교실 실습·발표용으로는 괜찮지만 진짜 서비스에서는 절대 이렇게 두면 안 됩니다. 맨 아래 **보안** 항목을 읽어주세요.

---

## 4단계 — 설정 키 복사하기

1. 좌측 **⚙️(톱니바퀴) 「설정」** 클릭 → 하위 메뉴가 펼쳐집니다
2. **「일반」** 클릭

> ⚠️ **"프로젝트 설정"이라는 항목을 찾지 마세요. 없습니다.**
> 콘솔 UI가 바뀌어서 예전의 "프로젝트 설정"이 지금은 **「일반」**입니다.
> 메뉴에는 일반 / 사용량 및 결제 / 통합 / 서비스 계정 / 데이터 개인 정보 보호 / 사용자 및 권한 / 알림 이 보입니다. **맨 위 「일반」**이 맞습니다.

3. 화면을 **맨 아래까지 스크롤** → **「내 앱」** 항목 → **`</>` (웹) 아이콘** 클릭
   - 안 보이면 스크롤이 부족한 것입니다. 프로젝트 정보가 먼저 나오고 앱 목록은 그 아래입니다.
4. 앱 닉네임: `jarada-web` → **「앱 등록」**
   - "Firebase 호스팅 설정"은 **체크하지 마세요** (지금은 불필요)
5. 화면에 이런 코드가 나옵니다. **`firebaseConfig = { ... }` 안쪽만** 필요합니다.

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyC7xK...",
  authDomain: "jarada-1234.firebaseapp.com",
  databaseURL: "https://jarada-1234-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId: "jarada-1234",
  storageBucket: "jarada-1234.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abc123"
};
```

---

## 5단계 — `firebase-config.js` 에 붙여넣기

프로젝트 폴더의 **`firebase-config.js`** 를 열고, `"여기에-붙여넣으세요"` 자리를 4단계에서 복사한 값으로 하나씩 바꿉니다.

> ⚠️ **`databaseURL` 이 없으면 절대 동작하지 않습니다.**
> 4단계 화면에 `databaseURL` 줄이 안 보인다면 **2단계(Realtime Database 만들기)를 안 한 것**입니다. 돌아가서 만들고 오면 그때 생깁니다.

---

## 6단계 — 확인하기

1. `서버켜기.ps1` 을 오른쪽 클릭 → **「PowerShell에서 실행」**
2. 열린 페이지에서 **학생용** → 이름 입력 → 시작하기
3. 할 일이 3개 보이면 **성공입니다** 🎉
4. Firebase 콘솔의 **Realtime Database → 데이터** 탭을 보면 방금 만든 학생이 들어와 있습니다

### 잘 안 될 때

브라우저에서 **F12 → Console** 탭을 열고 빨간 글씨를 확인하세요.

| 증상 | 원인 | 해결 |
| :--- | :--- | :--- |
| "서버 설정이 아직 안 됐어요" 화면 | `firebase-config.js` 가 안 채워짐 | 5단계를 다시 |
| `Can't determine Firebase Database URL` | `databaseURL` 이 비어 있음 | 2단계(Realtime Database)를 안 만든 것 |
| `permission_denied` | 보안 규칙이 잠겨 있음 | 3단계를 다시 (테스트 모드 30일 만료 확인) |
| 화면이 하얗게만 나옴 | 파일을 그냥 더블클릭함 | `서버켜기.ps1` 로 실행 |

## 학생 50명에게 나눠주기

지금은 파일이 내 컴퓨터에만 있어서 다른 기기에서 열 수 없습니다. 학생들이 각자 기기에서 접속하려면 **인터넷 주소가 필요합니다.** 가장 쉬운 두 가지입니다.

### 방법 A — Firebase Hosting (같은 콘솔에서 끝남)

터미널에서:
```
npm install -g firebase-tools
firebase login
firebase init hosting     # public 디렉터리를 . (점) 으로, SPA는 No
firebase deploy
```
→ `https://jarada-1234.web.app` 같은 주소가 나옵니다.

> ⚠️ 이 컴퓨터엔 **Node.js가 설치돼 있지 않습니다.** (제가 확인했습니다) 먼저 https://nodejs.org 에서 설치해야 `npm` 명령이 동작합니다.

### 방법 B — GitHub Pages / Netlify Drop

- **Netlify Drop**(https://app.netlify.com/drop): 폴더를 브라우저에 끌어다 놓기만 하면 주소가 나옵니다. **설치가 전혀 필요 없어 가장 빠릅니다.**
- GitHub Pages: 저장소에 올리고 Settings → Pages 에서 활성화

## 용량은 충분한가요?

무료 요금제(Spark) 한도와 이 앱의 실제 사용량입니다.

| 항목 | 무료 한도 | 학생 50명 기준 | 여유 |
| :--- | :--- | :--- | :--- |
| 동시 접속 | **100명** | 51명 (학생 50 + 선생님 1) | ✅ 절반 |
| 저장 용량 | 1 GB | 1명당 수 KB → 전체 1 MB 미만 | ✅ 넉넉 |
| 월 다운로드 | 10 GB | 실시간 구독을 감안해도 여유 | ✅ |

**동시 접속 100명이 유일하게 신경 쓸 한도입니다.** 50명이면 절반이라 안전합니다. 100명을 넘길 계획이면 그때 다시 상의하세요.

## 🔒 보안 — 발표 때 질문받으면 이렇게 답하세요

이 앱에는 **의도적으로 남겨둔 보안 구멍이 두 개** 있습니다. 몰라서 그런 게 아니라 2일이라는 기간에 맞춰 내린 결정입니다. 심사자가 물으면 솔직히 말하는 편이 훨씬 좋은 인상을 줍니다.

**1. 이름만으로 들어가는 건 "로그인"이 아닙니다**
비밀번호가 없으므로 누구나 남의 이름을 입력해 그 아이 정원에 들어갈 수 있습니다. 정확히는 **프로필 선택**입니다.
→ 제대로 하려면: Firebase Authentication (익명 로그인 또는 이메일 로그인)

**2. 보안 규칙이 열려 있습니다**
`.read: true, .write: true` 는 주소만 알면 누구나 전체 데이터를 읽고 지울 수 있다는 뜻입니다.
→ 제대로 하려면: 로그인을 붙인 뒤 규칙을 이렇게 좁힙니다.
```json
{
  "rules": {
    "jarada": {
      "students": {
        "$id": {
          ".read": true,
          ".write": "auth != null && auth.uid == $id"
        }
      }
    }
  }
}
```

**교실 실습·발표 범위**에서는 지금 상태로 충분합니다. 다만 **실명과 진짜 개인정보는 넣지 마세요.** 연습용 이름을 쓰시길 권합니다.
